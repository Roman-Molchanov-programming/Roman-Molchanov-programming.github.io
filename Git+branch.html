Branch-Commands

  • Создание бранча - git checkout -b название бранча (M-17-correct-attribute-order-for-meta-tag)
  • Названия доступных бранчей - git branch
  • Проверка всех бранчей - git branch -r
  • Проверка в изменениях в файлах - git diff (q - для выхода)
  • Выбор бранча - git checkout название бранча
  • Пуш бранча - git push -u origin название

Mergin blocked and pull request

workflow по работе над фичей Создать бранч с нормальным именем и закинуть его на сервер связав: 1. Git checkout main (Бранчи нужно делать строго от этого бранча)

2. Git pull.

Обязательно выполнять git pull после чекаута на main что бы подтянуть возможные чужые изменения.

3. Git checkout -b branchx. 4.!!!!!!!!!!!!!!!!!! Git push -u origin branchx. 5.Перед git add лучше всегда делать git diff. 6.В комите лучше указывать (M-20). 7.Сначала git push -u origin (Создание и связывание бранча с сервером) а только потом git push (Закидка кода на бранч).

Delete branch

На гитхабе нажали кнопку удалить бранч. Команда Delete branch на github не удаляет физический бранч с сервера а лишь ра зрывает связь между локальным бранчем и бранчем на сервере
