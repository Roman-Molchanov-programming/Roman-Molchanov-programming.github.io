Стандартные команды

  • git init - делает папку доступной для работы из гита
  • git status - показывает состояния файлов в рабочей директории и индексе
  • git commit - берёт все данные, добавленные в индекс с помощью git add, и
    сохраняет их слепок во внутренней базе данных(делает commit).
  • git add - добавляет содержимое рабочей директории в индекс (staging area)
    для последующего коммита
  • git diff - используется для вычисления разницы между любыми двумя Git деревьями.
    Это может быть разница между вашей рабочей директорией и индексом (собственно git diff),
    разница между индексом и последним коммитом (git diff --staged), или между любыми двумя коммитами (git diff master branchB).
  • git push - закидывает закоммиченную папку на сервер
  • git clean -xdf - для того чтобы удалить из гит уже удалленный файл или папку

Чтобы закинуть папку на гит нужно

  1. Зайти в нужную папку которая иницилизирована и связана с сервером.
  2. Добавить все с помощью git add --all или git add (.).
  3. Прежде чем делать коммит проверте что файлы добавлены (git add) с помощью (git status)
  4. Закомитить с помощью git commit -m "коментарий для коммита".
  5. Закинуть на github с помощью команды Git push.
  6. Прежде чем писать команду (git push) введите команду (git status), должно быть вот так

    7. Обратите внимание на красное выделение в скриншоте. Запушить можно сразу 2 файла и более.

Главная ошибка по которым файл может не попасть на гит

Файл может лежать не в нужной папке. Нужна папка - папка которая иницилизирована и связана с сервером.

Есть 3 типа репозиторий
  • Локальный (после коммита)
  • Удаленный(Remote) - (после пуша)
  • Имя которого совпадает с именем пользователя с прекрепленным в Конце
    Github.io. Таким образом github предоставляет возможность хостинга статических файлов.

Branch-Commands

  • Создание бранча - git checkout -b название бранча (M-17-correct-attribute-order-for-meta-tag)
  • Названия доступных бранчей - git branch
  • Перечисление всех бранчей на сервере - git branch -r(remote)
  • Проверка в изменениях в файлах - git diff (q - для выхода)
  • Выбор бранча - git checkout название бранча
  • Пуш бранча - git push -u origin название

Delete branch

На гитхабе нажали кнопку удалить бранч. Команда Delete branch на github не удаляет физический бранч с сервера а лишь ра зрывает связь между локальным бранчем и бранчем на сервере.
